### Section 5. AI 스킬 활용 전략 및 부서 차원 계획

**현재 AI 활용 현황**
- 각자 개인 생산성 향상 위주로 AI 활용 중
- 공통적으로 쓰는 것도 있고, 다르게 쓰는 것도 있음
- **성과 평가 항목에 AI 활용이 포함**됨 → 기록 필수

**Claude 스킬 활용 계획**
- **Claude 스킬**: 자주 쓰는 프롬프트를 스킬 문서화하여 공유 가능한 형태로 만드는 시스템
- Anthropic에서 스킬 시스템을 만든 이유: **공유용** (프롬프트 공유 시 동일 결과가 나오지 않는 문제 해결)
- **현재 시장에서 가장 쉐어러블(shareable)한 형태**

**2~3월 계획**
- 각자 한 영역씩 맡아서 **스킬 고도화** 진행
- 개인 생산성 → **팀/조직 전체 생산성**으로 무게 중심 이동
- 예시: 회의록 요약 스킬 (고유명사 교정 등 지속 업데이트 필요)

**n8n 활용**
- 파이프라인 자동화는 n8n으로 진행
- 파이프라인의 각 구성요소(생성형 AI 기능)는 Claude 스킬로 구현
- 슬랙 연동 시 슬랙 봇 등록 필요 (슬랙 어드민 요청하면 가능)

**GitHub 스킬 라이브러리**
- GitHub에 Claude 스킬이 많이 오픈소스로 공유되어 있음
- 팩스플, 모나스 브랜드 가이드라인, 캔버스 디자인 체인지, 로고 제너레이터, MCP 빌더 등 다양한 스킬 존재
- 폴더 복사하여 Claude에 적용 가능
- Cursor에서는 스킬 폴더를 만들어 AI 코디네이션으로 호출 가능

**로컬 RAG 구축 경험 공유**
- 스킬을 활용해 데이터/리포트 생성 후 **FAISS (페이스북 벡터 DB)**로 저장
- Chroma DB는 유료, FAISS는 무료
- 벡터 DB로 저장 시 AI 챗봇이 검색/참조하기 용이
- **로컬 RAG** 구축으로 필요 정보 검색 가능
- 회의록 대시보드도 RAG로 구축

**Google File Search API**
- 처음 업로드 시에만 비용 발생
- 단, **기업용 API 미출시** → 회사에서는 아직 사용 불가

### Section 6. 인디아 본부 마일스톤 리뷰 관련

**인디아 본부 프로세스 설명 회의 진행**
- 현일 님 요청은 **크리켓 관련 이슈만** 해당
- 다른 게임은 별도 요청 시 진행

- 두 분이 개발자 출신
- 전통적인 마일스톤 리뷰(검사)로 오해 → **의사결정 회의**임을 설명 필요

**크리켓 라이센스 관련**
- 의사결정 항목: IP(라이센스) 계약 - 크리켓 리그 정식 라이센스
- **게임 4개 동시 개발 중** → 의사결정 항목 클래리파이 요청 (4개 중 선택? 통합?)
- 비딩 타임 미확정 → **2028년까지 갈 수도** 있음
- **올해 마일스톤 리뷰 미진행 가능성** 높음 (내년 진행 권고)

**크리켓 라이센스 시장 상황**
- 인도에서는 크리켓이 국민 게임 → 라이센스 확보 시 큰 파이
- 현재 대부분 게임이 라이센스 없이 운영 중
- 연맹에서 라이센스 판매 본격화 예정 (2~3년 소요 예상)
