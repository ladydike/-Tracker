# [세션 제목: MR AI Study - 실무 활용 및 대시보드 개발 가이드]

## 1. 개요
* **분석 대상:** AI 업무 자동화 도구(Cursor, Claude, MCP 등) 및 대시보드 프로토타입 개발 사례
* **세션 목적:** 팀 내 AI 활용 현황 공유 및 바이브 코딩(Vibe Coding)을 통한 실무 생산성 향상 방법 전파

## 2. 상세 학습 내용 (주제별 정리)

### Section 1. [AI를 활용한 대시보드 개발 및 바이브 코딩]
* **기획 및 명세 작성:**
  - AI에게 막연하게 요청하기보다 **노션(Notion) 등을 활용해 상세 명세서를 작성**하고, 이를 MD 파일로 변환하여 AI에게 전달함으로써 정확도를 높임.
  - 비전공자의 경우 **"문과생/비전공자가 이해할 수 있는 단어로 설명해달라"**는 프롬프트를 추가하여 기술적 장벽을 낮춤.
* **실시간 대시보드 구현 사례:**
  - **Cursor(커서):** `Ctrl + L` 기능을 활용해 실시간 프롬프트 입력을 통한 라이브 코딩 진행.
  - **데이터 연동:** `데이터 브릭스(DataBricks)`와 SQL 쿼리를 연동하여 최신 정보를 실시간으로 조회하고 누적하는 테이블(`Product Details`) 설계.
  - **사용자 편의성:** 지난 분기 데이터 불러오기, 수정된 부분만 작성 기능 등을 통해 데이터 입력 효율 극대화.

### Section 2. [AI 도구별 특성 및 기술적 인사이트]
* **도구별 비교:**
  - **Claude(클로드):** 코드 작성 성능이 우수하며 `Canvas` 기능을 통해 HTML 결과를 즉시 확인 가능. **MCP(Model Context Protocol)** 표준을 제시하여 다양한 툴 연동에 강점.
  - **Cursor(커서):** 로컬 디렉토리 기반으로 파일 생성 및 수정이 용이하며, 에러 발생 시 에러 코드를 직접 입력하여 수정하는 방식이 효율적.
  - **GPT:** 코딩 성능이 클로드에 비해 다소 낮으며, 파일 다운로드 및 실시간 수정의 편의성이 부족함.
* **에러 핸들링 팁:**
  - **에러 코드 출력 필수:** AI에게 "에러 코드 분기 처리 및 테스트 코드 작성"을 명시적으로 요청하여 문제가 발생했을 때 원인을 즉시 파악할 수 있도록 함.
  - **최신성 체크:** AI가 간혹 구 버전의 API 모델을 사용하는 경우가 있으므로, 최신 모델 명칭(예: `Claude 3.5 Sonnet`)을 명확히 지정해줘야 함.

### Section 3. [인프라 및 확장성: 서버, DB, 클라우드]
* **서버와 클라이언트 구조:**
  - 문과생 버전 설명: PC(손님/브라우저)가 서버(이모)에게 요청하면 DB(주방 창고)에서 재료를 가져와 요리(애플리케이션)를 내주는 구조와 동일함.
  - **Azure(애저):** 파이썬 코드를 24시간 실행하거나 웹 앱 게시, 클라우드 DB 연동 등 확장성 있는 인프라 구축을 위해 활용.
* **데이터 브릭스 활용 이유:**
  - 로컬 DB 대신 데이터 브릭스를 사용하는 이유는 **전사적 데이터 공유 및 타 서비스와의 확장성** 때문임. 공동 작업 및 실시간 외부 공유가 필요한 데이터는 클라우드 DB 활용이 필수적임.

### Section 4. [MCP 및 워크플로우 자동화]
* **MCP (Model Context Protocol):**
  - AI가 외부 툴(컨플루언스, 슬랙 등)에 직접 접근하여 컨텍스트를 가져오거나 수정할 수 있게 해주는 프로토콜.
  - 마일스톤 리뷰 등의 컨텍스트를 학습시켜 AI가 팀원을 대신해 답변하게 하는 자동화 시스템 구축 목표.
* **슬랙 워크플로우(Chris App):**
  - 위키(Wiki)의 투두 리스트 중 체크되지 않은 항목을 매주 취합하여 알려주는 워크플로우 등을 통해 간단한 자동화 구현 가능.

## 3. Q&A 및 부연 설명
* **Q: API 키를 발급받을 때 비용 문제가 발생하지 않나요?**
  - **A:** API 키는 '식권'과 같으며 사용한 만큼만 비용이 발생함. 개인 학습용으로는 소액(0.08달러 수준)만 발생하며, 전사적으로는 오가니제이션 단위의 한도 제한(예: 500만 원)이 걸려 있어 안전함.
* **Q: 데이터 보안 문제는 없나요?**
  - **A:** 회사 기밀 데이터를 직접 AI 학습 데이터로 넣지 않는 한, API를 통해 오가는 데이터는 일반적으로 학습에 사용되지 않으므로 실습 수준에서는 큰 문제가 없음.

## 4. 요약 및 결론
* **핵심 관통 한 줄 요약:** AI는 단순한 답변 도구를 넘어 **명세 작성-구현-테스트-배포**의 전 과정을 지원하는 협업 파트너이며, 이를 위해 기초적인 인프라(Server, DB) 이해와 적극적인 '잡고 쓰기'가 핵심임.
