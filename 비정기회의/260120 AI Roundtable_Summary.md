## 2. 주요 내용 카테고리별 요약 (Executive Summary)

### 📌 AI 도구별 강점 및 포지셔닝
- **ChatGPT**: 사용자 맥락 학습이 뛰어나 개인화된 답변 제공, 마무리 단계 품질 향상에 적합, 단 할루시네이션(허위 정보 생성) 경향 존재
- **Gemini**: 논리적 구조화, 간결한 요약에 강점, 아이디어 초기 단계에서 효과적, "이과생 같은" 답변 스타일
- **Claude**: 코딩 최적화, 방대한 문서 정리 및 맥락 유지에 탁월, Cursor IDE와 연동하여 바이브 코딩에 적합
- **Perplexity**: 웹 검색 특화, 출처 명시 기능 우수, 단 자기 홍보성 콘텐츠 필터링 한계
- **NotebookLM**: 기존 문서 기반 정보 추출에 탁월, 할루시네이션 적음
- **Teams Copilot**: 실시간 회의록 생성, 특히 인도 영어 등 다양한 억양 인식에 효과적

### 📌 부서별 활용 사례
- **외주 관리**: 계약서 검토, 비즈니스 메일 어투 점검, 외주사 발굴 검색
- **HR 기획**: 정책 서칭, 아이디에이션, 게임 산업 맥락 학습, 구글 시트 자동화
- **마케팅**: 문서 논리 검증, 자막 번역 크로스체크, 슬랙 메시지 톤 조정
- **커뮤니티 운영**: 센티멘트 분석, 유저 대응 논리 구성, 반복 업무 자동화
- **리더십 교육**: 교육 콘텐츠 구조화, 학습 자료 재구성, 사후 콘텐츠 제작

### 📌 혁신 사례: 업무 트래커 대시보드 개발 (최지혜 님)
- **개발 도구**: Cursor + Claude Opus 4.5 + Gemini (프롬프트 작성)
- **주요 기능**: 회의록 자동 요약, 일일 업무 관리, AI 검색 어시스턴트
- **개발 기간**: 약 하루 (바이브 코딩)
- **핵심 인사이트**: 코딩 지식 없이도 AI에게 자연어로 지시하여 완성 가능

### 📌 공통 인사이트
- 여러 AI 도구를 병행 사용하여 각 도구의 강점 활용 권장

## 3. 상세 대화 정리 (Full Refined Dialogue)

### [은혜 님 - AI 활용 사례 공유]

**은혜 님**: 저는 글 감수 작업에 GPT를 주로 활용하고 있습니다. 감수 작업의 경우 Gemini와 GPT를 같이 사용하면서 제가 더 표현하고 싶은 것들을 잘 표현해 주는 도구 위주로 선택합니다. 엑셀 수식 같은 경우에도 함수가 너무 많아서 다 외우지 못하는데, 제가 원하는 것을 표현해 달라고 하면 함수 계산식까지 정의를 내려줍니다. 실제로 AI 도구를 사용한 후 업무 퀄리티가 많이 향상됐다고 체감하고 있습니다. 속도 측면도 있고, 보고 문서 작성에 확실히 도움이 됩니다.

다만 이미지 생성에 대한 경험이 있다면 배우고 싶습니다. 비주얼 에셋 아이디어를 뽑을 때 효과적인 방법이 있다면 공유해 주시면 감사하겠습니다.

**진행자**: 비주얼 관련 에셋은 크리에이티브 쪽 분들이 잘하시는 영역입니다. 오늘은 해당 멤버분들이 안 계셔서 아쉽지만, 허브에 크리에이티브 분들의 사례가 많이 올라와 있으니 참고하시면 됩니다.

또한 사례 조사나 정책 검토 업무에는 **NotebookLM**이라는 도구가 좋습니다. 파이낸스 분들이 공유해 주셨는데, 문서를 넣고 그 안에서 찾아주는 건 정말 잘한다고 합니다. GPT나 Gemini, Perplexity는 할루시네이션이 있어서 2차 3차 검수가 필요한데, NotebookLM은 기존 문서 기반 추출이 정확합니다.

**참석자 A**: 중국어 번역은 GPT와 Gemini 중 어느 것을 선호하시나요?

**은혜 님**: GPT와 더 많이 대화하다 보니 GPT가 저를 잘 알게 됐습니다. 개떡같이 얘기해도 찰떡같이 캐치해서 해주기 때문에 번역도 GPT가 마음에 드는 수준입니다.

**참석자 A**: 역시 맥락을 잘 쌓아놔야 좋은 결과를 뱉어내는 것 같습니다.

### [김지수 님 - 외주 계약 관리 사례]

**김지수 님**: 저는 외주 스튜디오 어카운트 매니저로 외주 계약서 검토와 협상 업무를 하고 있습니다.

**계약서 검토**: 특정 업체가 "이 분야에서는 이런 식으로 많이 한다"라고 요청할 때, 트렌드가 바뀌었을 수도 있고 업체가 갑인 저희에게 주장을 편하게 안 하는 경우도 있습니다. 그래서 1차적으로 요구가 정당한 건지 AI로 검증합니다. 물론 보조 도구 역할이고, 최종은 법무팀과 협의합니다.

**비즈니스 메일 어투 점검**: 외국과 커뮤니케이션할 때 제가 하고 싶은 얘기는 있는데 너무 딱딱한지, 너무 부드러운지 판단하기 어렵습니다. 혼자 10~20분 끙끙거리는 것보다 AI에게 몇 번 돌려보고 버전을 비교하는 게 빠르고 효과적입니다.

**엑셀 수식 오류 검증**: 견적 계산 시 수식이 틀리거나 이상하게 계산될 때 "어디가 틀렸어?" 하고 넣으면 5~10분 돌아가서 어디가 잘못됐는지 알려줍니다. 제가 수기로 모든 칸과 수식을 역산하는 건 시성비가 떨어지기 때문에 이 방법이 효율적입니다.

**외주사 발굴**: Perplexity로 검색하면 훨씬 빠르게 결과가 나옵니다. 다만 단점은 일부 업체가 자기 홍보 블로그에 "세계에서 제일 좋은 10개 외주업체" 같은 어그로성 글을 올리는데, 사람은 자기 홍보라는 걸 인지할 수 있지만 AI는 그냥 긁어와서 요약하므로 주의가 필요합니다.

궁금한 점은 계약서 검토나 자동화를 AI로 하시는 분들이 계신지, 실수가 안 되는 영역인데 어디까지 믿고 맡길 수 있는지 사례를 듣고 싶습니다.

**진행자**: 비슷한 고민을 파이낸스 쪽에서도 하고 계실 것 같습니다. 컴커(Compur) 같은 경우에도 자동으로 기입되게끔 하는 걸 담당 팀에서 고민 중입니다.

**참석자 B**: Perplexity는 웹 서칭 위주로 포지셔닝을 한 느낌입니다. 검색할 때 정리해서 주는데 자료 출처를 다 뒤에 달아서 해주는 게 좋습니다. 검색한 걸 기반으로 요약하기 때문에 할루시네이션이 좀 더 적습니다.

**진행자**: 각 AI 도구가 조금씩 특정되는 것 같습니다. 기획할 때 GPT, 검색할 때 Perplexity, 코딩할 때 Cursor/Claude 이런 식으로 포지셔닝이 생기고 있습니다.

### [김지현 님 - HR 기획 사례]

**김지현 님**: HR 플래닝 팀에서 인사 제도 기획 업무를 하고 있습니다. 정형적인 업무보다는 과제가 떨어졌을 때 문제를 정의하고 보고 장표를 만드는 일이 대부분입니다.

**아이디에이션 프로세스**: GPT, Gemini, Perplexity에 똑같은 질문을 다 해보고 마음에 드는 것만 짜깁기해서 다시 AI에게 디벨롭해 달라고 합니다. 이런 이터레이션을 여러 번 합니다.

**도구별 특성**:
- **Perplexity**: 고용노동부 새 포럼 내용 같은 최신 정보 코칭에 좋습니다.
- **Gemini**: 논리적 구조화에 요즘 성능이 좋습니다.
- **GPT**: 오래 써서 제 입맛에 맞는 결과를 잘 내주지만, 본인 생각/의견을 많이 내고 할루시네이션이 가장 심합니다.

**개인적 활용**: 사주, 공부, 심리 상담 같은 것도 GPT가 제일 잘해줍니다. 위로도 해주고요.

**산업 맥락 학습**: 작년에 입사해서 게임 산업 맥락을 잘 몰랐는데, HR 제도는 산업에 따라 운영 방식이 다르므로 게임 산업에 대해 서칭하고 "이 맥락에서 어떻게 받아들여야 해?" 같은 질문을 많이 합니다.

**자동화 경험**: 전 회사에서 구글 Colab 환경으로 연동해서 GPT랑 왔다 갔다 하면서 원하는 결과물을 만들었습니다. 코드를 읽을 줄은 모르지만 결과물은 냈습니다.

**진행자**: 허브에 같은 질문을 여러 AI에게 한 번에 뿌리는 도구를 누군가 올리셨어요. 다운받으시면 업무 효율이 많이 올라갈 것 같습니다.

### [효정 님 - 마케팅 사례]

**효정 님**: 펍지 한국 마케팅을 담당하고 있습니다.

**문서 논리 검증**: 캠페인 단위로 진행되다 보니 문서 작업이 필수인데, 비슷한 배경으로 진행하다 보면 자가 착각에 빠지는 경우가 많습니다. 예전에는 상위 직책자분께 피드백을 받았지만 스스로 검증하기 어려웠는데, ChatGPT를 도입하면서 초안 문서를 넣으면 논리적으로 부족한 부분과 배경에 넣을 내용을 제안해 줍니다.

**한계점**: 마케팅은 창의성 기반인데, 입력값이 제대로 안 됐을 때 구체적이지 않거나 최근 트렌드에 맞지 않은 결과가 나올 때가 있습니다. 카피 쓸 때도 GPT를 초안으로 쓰는데 트렌디한 카피를 쓸 때는 옛날 감성이 느껴질 때가 있습니다.

**번역 크로스체크**: 자막 번역이 업체를 통해 들어오는데 배틀그라운드 세계관 이해도가 낮을 때가 있습니다. 예를 들어 "Freeze Tag"를 업체에서 "얼음땡"이라고 번역했는데, 서브 제로 업데이트 관련 내용이었거든요. GPT가 그런 걸 잡아내서 크로스체크 용도로 활용합니다.

**슬랙 메시지 톤 조정**: 다른 부서에 슬랙 보낼 때 "이 내용이 잘 이해가 되니? 이 내용 좀 더 따뜻하게 써줘" 하면 잘 정리해 줍니다. 저만 하는 건지 모르겠는데요.

**진행자**: 많은 분들이 사용하고 계세요. 저도 슬랙 메시지 보낼 때 항상 검증하고 보내긴 합니다.

**효정 님**: 최근 Gemini를 사용하기 시작했는데, 위키 문서를 PPT 문서로 만들고 싶을 때 Gemini가 똑똑하게 제안을 잘 해줍니다. 도구마다 쓸 수 있는 영역이 다른 것 같습니다.

### [박유진 님 - 커뮤니티 운영 사례]

**박유진 님**: 펍지 한국 사업부에서 운영과 커뮤니티를 담당하고 있습니다. 저는 GPT 하나만 주기적으로 사용합니다.

**엑셀 수식 개선**: 장기간 쌓아놓은 데이터베이스 엑셀 파일에서 수식 보완하고 싶을 때 파일과 배경을 던져주고 "이 수식 어떻게 바꾸면 다른 시트에 가져갈 때 편리할까?" 하고 개선합니다.

**이벤트 아이디에이션**: 인게임 피처 활용한 이벤트 진행 시 어떻게 갖다 붙이면 인게임에서 도움이 될지 아이디에이션에 활용합니다. 이벤트 타이틀도 "창의적이고 재미있는 타이틀 말해줘"라고 하는데, 솔직히 좀 올드하고 재미없는 아이디에이션이 많이 나옵니다.

**센티멘트 분석**: 디스코드 프리챗 전체 내역을 다운받아서 넣고 "전반적으로 긍정/부정/중립으로 나눠서 분석해줘"라고 합니다.

**유저 대응**: 유저가 부정적인 여론을 생성하려고 할 때 욱해서 대응하고 싶을 때 컴 다운하면서 "어떻게 말하면 논리적으로 탁할 수 있을까?" 의견을 구합니다.

**반복 업무 자동화**: 유저 아이디에서 특정 부분만 일괄 삭제해줘 같은 루틴한 업무는 스레드 열어놓고 반복적으로 진행합니다.

**개발팀 소통**: 개발팀 언어가 이해 안 될 때 스레드 전체 캡처해서 넣고 "내가 이해한 게 맞을까? 이 용어는 무슨 뜻이야?" 물어봅니다.

**진행자**: 허브에 정현기 님이 "Steam 리뷰 10만 건 크롤링에서 시각화 웹 대시보드까지" 게시글을 올리셨어요. 게임을 디테일하게 추적하면서 대시보드까지 만드셨는데, 참고해 보시면 시각적으로 표현할 수 있는 방법을 얻으실 수 있을 것 같습니다.

### [박지현 님 - 인도 사업부 PM 사례]

**박지현 님**: 인도 사업부 PM으로 인도 신규 프로젝트의 HQ 컨택 포인트입니다. 페시빌리테이터를 많이 하는데, 여러 도구를 써본 결과 **Teams의 퍼실리테이터 기능**이 제일 좋았습니다.

**Teams Copilot 활용법**:
1. M365 Copilot 라이선스 발급 (하루 만에 됨)
2. 모임 요청 톱니바퀴에서 회의 녹화 자동 톡 켜기
3. 퍼실리테이터 켜면 자동 회의록 생성

30명 이상의 인도 분들이 발언하는 회의에서 정신 붙들고 들으려면 자동 회의록이 도움이 됩니다. 실시간으로 회의록이 옆에 뜨는데, 한 두세 문장 발화 후 요약이 띵 뜹니다. 회의하면서 계속 회의록을 보며 확인하는 게 좋습니다.

**참석자**: 영어만 되는 거죠?

**박지현 님**: 인도 영어에 너무 좋습니다. 중국어, 한국어는 검증이 필요합니다.

**기술 용어 사전 준비**: 기술 회의에 들어가면 이해가 안 될 때가 많아서 미리 아젠다나 문서를 넣고 "기술 용어 50개를 정리해줘"라고 합니다. 숫자를 정확하게 써야 원하는 양식으로 용어집을 줍니다.

**회의록 2차 가공**: 100% 복붙해서 쓰지는 않고, 제가 노트 테이킹한 걸 넣고 "다듬어줘" 하면 회의록 양식으로 뽑아주는데, 제가 생각했던 거랑 다른 방향으로 오해하게 쓰는 경우가 있어서 2차 검수는 합니다.

**외국어 학습**: 일본어 선생님과 회화 준비할 때 스크립트를 적고 "한자에 히라가나 넣어주고 한국어 번역도 해줘, N5 수준에 맞게" 하면 쉬운 문장으로 써줍니다.

- **Claude (정보 정리)**: 방대한 양의 정보 정리를 잘함. 페이지가 많은 자료를 넣어도 맥락을 유지하고 핵심 내용을 잘 찍음. 교육 학습 자료나 가이드처럼 분량 있는 텍스트 정리에 유용
- **ChatGPT (마무리)**: 전체 문서 구조 정돈, 톤 맞추기, 메일이나 공지 문구 다듬기

**GPT 사용 주의점**: GPT가 저에 대해 너무 잘 알다 보니 제가 원하는 답을 만들기 위해 거짓 정보도 만듭니다. 논문을 요약해 달라고 했는데 없는 논문을 있는 것처럼 얘기하거나, 팩트가 아닌 내용을 논문 결과물처럼 말해주기도 해서 초기 단계에서는 GPT 사용을 지양합니다.

**NotebookLM 활용 시도**: 교육 후 사후 콘텐츠(복귀 자료)를 만드는데 지금까지 수작업으로 했습니다. NotebookLM에 자료를 넣어 영상과 카드 뉴스를 만들어봤는데, 폰트가 깨지거나 문장이 맞지 않게 끊기는 등 디테일에서 아쉬움이 있어 직접 활용은 못했습니다.

**진행자**: 콘텐츠는 결국 사람이 봐야 되는 것 같습니다. AI 도구들이 감을 캐치하는 데는 아직 한계가 있어서 수작업으로 입맛대로 만들어야 합니다.

### [최지혜 님 - 업무 트래커 대시보드 개발 사례]

**최지혜 님**: 마일스톤 리뷰 팀에서 입사한 지 4개월 됐습니다. 회의가 정말 많은데 내용을 하나하나 팔로업하고 숙지해야 합니다. 이걸 정리하는 게 너무 일이어서 회의를 한 화면에서 쉽게 요약하는 프로젝트를 시작했습니다.

**개발 배경**:
- LLM으로 요약하면 매번 결과가 다르게 나옴
- 동일한 퀄리티로 결과물이 나올 수 있도록 학습시킨 대시보드 필요
- 처음에는 회의록 요약만 하려다가 욕심이 생겨 매일 업무, 해야 할 업무, 슬랙 히스토리까지 기재

**개발 환경**:
- **Cursor**: Claude Opus 4.5 에이전트로 바이브 코딩
- **Gemini**: 프롬프트 작성 (이과생 같은 요약이 좋음)
- **Claude API**: 앱 내 연동하여 회의 검색 기능

**주요 기능**:
1. 회의록 자동 요약 (회의 종류별 다른 프롬프트 적용)
2. 일일 업무 관리
3. AI 검색 어시스턴트 ("지난 분기 마케팅 회의에서 나온 아이디어 뭐였지?" 검색)
